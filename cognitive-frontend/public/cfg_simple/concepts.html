۱. گراف و مسیر

  • Path: دنبالهٔ گره‌ها — مثلاً [1, 3, 4, 5, 6]
  • Test Path: مسیری که یک تست در CFG طی می‌کند
  • Tour: پیمایش یال/گره · Sidetrip: انحراف موقت از مسیر اصلی
  • SESE: Single-Entry Single-Exit — هر بلوک یک ورودی و یک خروجی

۲. معیارهای پوشش ساختاری

معیارTest Requirementدر پروژه
Node Coverage (NC)هر گره ≥ ۱ بار۱۴ تست
Edge Coverage (EC)هر یال ≥ ۱ بار۹ تست + جدول coverage.html
Edge-Pair (EPC)هر جفت یال مجاوربا PPC پوشش می‌شود
Prime Path (PPC)هر prime path ≥ ۱ بار۶ تست + جدول PPC

۳. Subsumption (شمول)

NC ⊂ EC ⊂ EPC ⊂ PPC

پوشش قوی‌تر، پوشش ضعیف‌تر را شامل می‌شود. PPC برای حلقه‌ها مناسب‌تر از EC است.

۵. Data Flow Coverage

  • Def: جایی که متغیر مقدار می‌گیرد
  • Use: جایی که مقدار خوانده می‌شود
  • DU pair: (گرهٔ def, گرهٔ use)
  • DU path: مسیر def-clear از def تا use
معیارشرط
All-Defs (ADC)هر def ≥ ۱ use
All-Uses (AUC)هر DU pair ≥ ۱ بار
All-DU-Paths (ADUPC)هر DU path ≥ ۱ بار

جدول کامل: coverage.html

ADC ⊂ AUC ⊂ ADUPC
