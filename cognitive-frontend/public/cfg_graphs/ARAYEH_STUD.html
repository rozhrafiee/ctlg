۴ ایراد معماری — بررسی و اصلاح

۱. تناقض نوع داده (Data Interface)

نگرانی: مسیر ۲ «ایندکس» برمی‌گرداند؟ مسیر ۱ «آیتم»؟

واقعیت کد: هر سه مسیر list[dict] برمی‌گردانند.

indices = linear_search(items, query)
return [items[i] for i in indices]   # تبدیل به آیتم

برچسب اشتباه گراف اصلاح شد → AS2b: return [items[i] for i in indices]

۲. مرتب‌سازی دوبار (Redundancy)

واقعیت: فقط در مسیر binary: bubble_sort برای پیش‌شرط جستجو، سپس PC3 مرتب‌سازی نهایی.

trade-off آموزشی — در production فقط linear استفاده می‌شود (بدون مرتب‌سازی کمکی).

۳. جریان متادیتا (Metadata)

search_algorithm و sort_algorithm از پارامتر ورودی بعد از PC1 می‌آید، نه از _apply_search.

در گراف: خط چین بنفش از PC1 به PC4 اضافه شد.
